Filename Audit Checklist

The audit should be boring and strict. It is there to catch accidental mess before human review.

  • Reject filenames with spaces, dates used as version numbers, or local machine names.
  • Flag words that imply private scope, such as secret, internal, prod, or personal.
  • Require lowercase kebab-case for Markdown documents.
  • Check examples for placeholder naming, not real hostnames or account names.

Publication Review Prompt

Use this as the prompt body for prompts/publication-check.md. It is intentionally specific about outputs.

Markdown prompt 
Review this repository for public publication readiness.

Focus on:
1. Filenames and paths that imply private infrastructure.
2. Markdown examples that include real identifiers.
3. Screenshots, terminal output, or diffs that reveal local state.
4. Prompt files that encourage unsafe disclosure.
5. Documents that belong in a private runbook instead of this public repo.

Return:
- Blockers
- Warnings
- Safe-to-publish notes
- Suggested edits by file path

Run it after mechanical checks and before committing publication-oriented changes. The prompt should produce a review report, not silently rewrite the repo.
